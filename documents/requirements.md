# &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; Требования к веб-сервису "Доктор-онлайн"
## Идея проекта:

Нужно разработать сайт, с помощью которого будет осуществляться онлайн-запись к различным специалистам в различных мед.учереждениях города.
### Общее представление:
Многостраничный сайт в отенках зеленого.
Сервис должен эстетично выглядеть даже в старых версиях различных браузеров.
Сайт предоставляет возможность просмотреть свободное для записи на прием время различных специалистов (режим "гостя", без авторизации) или же записаться на прием, используя личный кабинет (должна быть реализована возможность регистрации соц.сети). Можно выбрать мед.учереждение, профиль специалиста и, по желанию, конкретного врача из работающих в данном учереждении (например, если работает несколько терапевтов - можно по фамилии выбрать "своего").
Можно заказать обратный звонок из больницы для более детальной консультации. Можно отменить запись на прием в личном кабинете.

#### Начальная страница:
Разворачивающиеся списки выбора города и находящихся в нем больниц (желательно, чтобы город посетителя определялся автоматически), календарь, список врачей.
После выбора города, больницы и дня посещения появляется расписание приема врачей(свободное для записи время подсвечивается).

#### Личный кабинет:
В личном кабинете отображаются записи пользователя к врачам. Есть возможность отменить запись.
Возможность включить напоминание о приеме с заданным сроком (за день до посещения, за 3, 5 и т.д.)

### Специфические требования:
##### Корректность:
* действительно решает задачу, для которой создано;
* разработано в соответствии с формальными правилами проектирования программного обеспечения.

##### Практичность:
* сервис интуитивно понятен для любого среднестатистического пользователя (всплывающие подсказки, понятные названия кнопок);

##### Эффективность:
* автоматическое заполнение данных пользователя из соц.сети или личного кабинета (имя, фамилия, номер страхового полиса и т.д.);
*  кэширование данных;

### Критерии качества:

##### Целостность: 
* пользователь вводит личные данные, поэтому необходимо обеспечить их конфиденциальность;

##### Живучесть:
* сервис не прекращает работу даже при некорректно введенных данных, сообщает об ошибке и предлагает ее устранить;

##### Понятность:
* код должен быть понятным и удобным для сопровождения при последующих модификациях.
